# 職をつぐ者 〜ジパング復興編〜

ドラゴンクエストビルダーズへのオマージュとして制作した、ブラウザで遊べる
「ブロックメイクRPG」の MVP です（three.js + Vite / WebGL）。

DQ3後の上の世界・ジパングを舞台に、ダーマ神殿崩壊で職を失った人々のもとへ
流れ着いた神官見習いが、ビルダーの書を手に町を復興していきます。
企画の全体像は [assets/dqb3_game_plan.md](assets/dqb3_game_plan.md) を参照。

## 起動方法

```powershell
npm.cmd install
npm.cmd run dev
```

表示されたローカルURL（通常 `http://localhost:5173/builder_sprite_lab/`）をブラウザで開いてください。
Windows では `start-dev.bat` でも起動できます。`npm run build` で `dist/` に出力され、
main ブランチへの push で GitHub Pages に自動デプロイされます（`.github/workflows/deploy.yml`）。

## 操作方法

| 入力 | 動作 |
| --- | --- |
| `W A S D` / 矢印キー | 移動（8方向） |
| `Space` | ジャンプ |
| 左クリック | 叩く（採掘・攻撃）／ブロック設置／種まき／食べる |
| 右クリック / `F` | 調べる・話す（作業台・ベッド・祭壇・住民など） |
| `1`〜`8` / ホイール | ホットバー選択 |
| `E` | もちもの・クラフトメニュー |
| `B` | 職業の書 |
| `Q` / `[` / `]` | カメラ回転（45°刻み・8方向のみ） |
| `M` | サウンド ON/OFF |
| `Esc` | ポーズ（セーブ） |

## ゲームの流れ（MVP: 約30〜60分）

1. 浜辺に漂着し、ビルダーの書と出会う
2. 木材を集めて作業台を作り、設計図どおりに小屋を建てる → 住民が加入
3. 畑を作り、住民と一緒に稲を育てて収穫 → **農民の書**が復活
4. 北の山で銅を掘り、武器・防具・柵・罠で防衛のそなえ
5. 夜の防衛戦：スライム・おおこうもりの群れ＋ボス「書喰いのスライム」から職業の旗を守る
6. 山の聖地に「職業の社」を建て、祭壇に祈りをささげるとエンディング → 以降は自由建築

昼夜サイクルがあり、拠点の外では常時モンスターがわきます（夜は増加）。
倒すと素材を落とし、装備強化（こん棒→石の剣→銅の剣、こうもりのマント等）に使えます。
セーブは localStorage（オートセーブ＋ポーズ画面から手動）。

## 実装の構成

```
src/
├─ main.js            ゲーム統合（ループ・入力・アクション・カメラ）
├─ core/              input.js（キー＋マウス）, utils.js（RNG・8方向変換）
├─ world/
│   ├─ blocks.js      ブロック定義＋テクスチャアトラス（全てコード生成）
│   ├─ world.js       ボクセルデータ・島の地形生成・DDAレイキャスト・セーブ差分
│   ├─ mesher.js      チャンク単位のメッシュ結合（不可視面カリング）
│   └─ props.js       設置物メッシュ（作業台・扉・たいまつ・稲・鳥居など）＋光源プール
├─ entities/
│   ├─ rig.js         2.5Dキャラリグ（8方向スプライト＋3D手足）※試作ラボから移植
│   ├─ player.js      主人公（自由移動・ジャンプ・水泳・装備）
│   ├─ villagers.js   住民4人（追従・農作業・戦闘参加・会話）
│   ├─ monsters.js    スライム／おおこうもり／ボス（8方向ビルボード2ポーズ切替）
│   └─ entity.js      AABB×ボクセル物理・アイテムドロップ
├─ game/              items.js, quests.js, blueprints.js, defense.js,
│                     daynight.js, save.js, sfx.js（WebAudio合成SFX）
└─ ui/ui.js           HUD・会話・メニュー・設計図パネル・タイトル・エンディング
```

### キャラクター表現（2.5D）

- **人間型**（主人公＋住民4種）: 頭と胴は 8方向スプライトシート（4×2, 128px セル）の
  UV窓切り出しを薄い Box の前面に貼り、腕・脚・道具は 3D トゥーンメッシュ。
  歩行スイング・攻撃モーション・向きごとの手足レイアウトは試作ラボの調整値をそのまま使用。
- **非人間型**（スライム・おおこうもり）: 8方向ビルボードで `idle_a` / `idle_b` の
  2ポーズを交互切替。スライムは跳ねの伸縮、こうもりは浮遊バウンスを追加。
  ボスは同シートの色替え＋拡大で表現。
- カメラは固定クォータービューを45°刻みで回転（キャラが8方向画像のため8方向限定）。
  視線が木の葉などで遮られると自動でズームインします。

### ワールド

- 64×64×24 のフルボクセル。掘削・設置自由（銅鉱石は石のハンマーが必要）。
- ブロックテクスチャ・アイテムアイコン・UIはすべてコード生成（画像アセットはキャラのみ）。
- 島の地形（浜・草原・西の森・北の火山風の山・拠点用の平地）はシード固定の手続き生成。
  セーブには生成後の差分のみを保存します。

## 既知の改善余地

- 住民の経路探索は直進＋ジャンプのみ（複雑な地形で引っかかることがある）
- 防衛戦の敵AIは旗直行型。柵・扉は攻撃で壊されるが、地形は壊さない
- 効果音は WebAudio の簡易合成。BGMなし
- モバイル（タッチ）未対応

## 旧・検証ラボについて

このリポジトリは元々 2.5D キャラ表現の検証用ラボでした。検証コードは本ゲームの
`src/entities/rig.js` に一般化して取り込み済みです。旧アセット
（`assets/player_image.png`, `assets/player_body_8dir.png`）と生成スクリプト
（`tools/keygreen.mjs`）は参考のため残しています。
