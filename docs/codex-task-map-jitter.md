# 指示：連続移動時にマップが小刻みに揺れる問題の修正

## 対象
`src/main.js`（Three.js / 単一ファイル構成。`renderer.setAnimationLoop` で毎フレーム `frame(delta)` を実行している）

## 課題（症状）
キャラクターをキー押しっぱなしで連続移動させると、追従するカメラ越しに
**マップ（背景）が一定速度でスクロールせず、小刻みに速くなったり遅くなったりして揺れて見える**。
プレイ中に軽い不快感がある。連続移動が滑らかに等速スクロールするようにしたい。

## 原因の仮説

### 仮説1（主因）：1マスごとにイージングを再スタートし、速度がパルス状になる
- `updateMovement`（`src/main.js:450` 付近）で、1マス移動を `MOVE_DURATION = 0.22s` の
  スムーズステップ `eased = t*t*(3-2t)` で補間している。
- この曲線は t=0 / t=1 で速度がほぼ 0。連続移動でも `t>=1` で `tryStartMove()` を呼んで
  **毎マスこの曲線を最初からやり直す**ため、キャラ速度が `0→ピーク→0→…`（約 4.5 回/秒）と脈打つ。
- カメラはキャラ位置を目標に追従するので、マップのスクロール速度も同じリズムで変動する。
  これが「小刻みな揺れ」の本体。

### 仮説2（増幅要因）：カメラ追従の平滑化がフレームレート依存
- `updateCamera`（`src/main.js:598`）で `camera.position.lerp(desiredPosition, 0.12)` と、
  **delta を掛けない固定係数**の指数スムージングを使用している。
- フレームレート依存で挙動が変わり（フレーム落ちでガクつく）、かつ常時ラグがあるため、
  仮説1のパルスを「カメラとキャラの相対オフセットの揺れ」として増幅する。

### 仮説3（副次）：高低差を跨ぐ際の縦揺れ
- 移動の Y も `moveEnd.y`（`src/main.js:445`）に行き先タイル高さを入れて線形補間しており、
  段差を跨ぐとカメラ注視点 Y が上下してマップが縦に跳ねる。

### 仮説4（副次）：斜め移動が約 1.41 倍速い
- 斜めも同じ `MOVE_DURATION` で √2 の距離を進むため方向で速度が変わる
  （ジッターではないが等速感を損なう一因）。

## 依頼する修正（優先度順）

1. **【最優先】連続移動時のイージング継ぎ目を解消する。**
   キーを押し続けて次マスへ連続する場合、マスの継ぎ目では ease-in/ease-out を入れず
   **等速で繋ぐ**。実装案：移動開始時のみ ease-in、最終停止マスのみ ease-out を適用し、
   途中マスは線形にする（＝「次の入力があるか」で停止マスか継続マスかを判定して補間カーブを
   切り替える）。あるいは連続中はマス境界をまたいで進捗を持ち越す方式でもよい。

2. **カメラ平滑化を時間ベースにする。**
   `camera.position.lerp(desiredPosition, 1 - Math.exp(-k * delta))` のように delta 基準の
   減衰へ変更し、フレームレート非依存にする（`k` は現状の体感に近い値を選定）。
   `updateCamera` は `delta` を受け取れるようシグネチャ変更可。

3. **（任意）高低差の縦補間を別途スムージング**し、段差跨ぎの縦揺れを軽減する。

4. **（任意）斜め移動時は `MOVE_DURATION` を √2 倍**にして実効速度を等速化する。

## 制約・確認事項
- 既存の操作感（1 マスグリッド移動、8 方向スプライト切り替え、`[` / `]` でのカメラ回転）は維持する。
- 当たり判定・グリッドスナップ（`snapPlayerToGrid`）・影位置更新（`updateShadowPosition`）の整合は崩さない。
- まず仮説1の修正だけで揺れがどの程度収まるか確認し、その後に 2 以降を適用して効果を比較できるとよい。

## 受け入れ基準
- キーを押しっぱなしにしたとき、マップが等速で滑らかにスクロールし、
  毎秒数回の速度変動（揺れ）が体感で消えていること。
- フレームレートが変動してもカメラ追従の挙動が大きく変わらないこと。
